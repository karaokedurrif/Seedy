## 📝 LESSONS LEARNED

1. **Docker Image Selection Critical**
   - Custom images may have hardcoded entrypoints → use official `vllm/vllm-openai:latest`
   
2. **GPU Memory Calculation**
   - Shared GPU: calculate (Total - Existing) before setting `--gpu-memory-utilization`
   - Ollama 31GB + vLLM 18GB = 49GB (safe with 128GB total)

3. **Unified Memory Architecture (GB10)**
   - `nvidia-smi` shows "N/A" for memory — normal, not a bug
   - Use `docker stats` for container memory tracking

4. **First Inference Latency**
   - 10-20× slower than subsequent (torch.compile + CUDA graphs)
   - Expected behavior, not a performance issue

5. **Container Naming**
   - Docker Compose service name ≠ container name
   - Service: `vllm-coder`, Container: `vllm-coder` (check with `docker ps`)

6. **Health Check Best Practices**
   - Use HTTP status code check, not body parsing
   - Include inference test (simple "2+2") to validate model
   - Log to file for troubleshooting history
