## 🖥️ Test 2: Openclaw SOC — System Troubleshooting

**Setup:**
- Openclaw miniPC (192.168.20.54) configurado → vLLM endpoint
- Antes: qwen2.5:7b-instruct (calidad 6-7/10)
- Ahora: qwen2.5-coder-32b (target 8-9/10)

**Tests:**

### 2.1 CPU Spike Diagnosis
```
Query: "CPU 95% constante en servidor Apache. ¿Qué verificar primero?"
Expected:
- Check procesos con top/htop (apache2 workers)
- Verificar logs acceso (DDoS?)
- Revisar módulos Apache cargados
- Check queries lentas PHP/MySQL
- Verificar límites MaxRequestWorkers
Quality Target: 8/10
```

### 2.2 Security Breach Detection
```
Query: "10 intentos de login SSH fallidos desde IP externa. ¿Siguiente paso?"
Expected:
- Verificar /var/log/auth.log para IP origen
- Check fail2ban status
- Banear IP manualmente con iptables si no está fail2ban
- Revisar otros accesos exitosos de esa IP
- Considerar cambiar puerto SSH
Quality Target: 9/10
```

### 2.3 Disk Space Emergency
```
Query: "/var/log/ ocupa 80GB de disco. ¿Cómo identificar logs grandes y limpiar?"
Expected:
- du -sh /var/log/* | sort -h
- Identificar logs > 1GB
- truncate -s 0 para logs no críticos
- logrotate config check
- Considerar eliminar logs antiguos (.gz)
Quality Target: 8/10
```

### 2.4 Memory Leak Investigation
```
Query: "Uso de swap al 90% en servidor MySQL. ¿Diagnosticar causa?"
Expected:
- free -h para ver RAM/swap
- top ordenado por memoria (%MEM)
- mysqladmin processlist (queries pesadas)
- Check innodb_buffer_pool_size vs RAM
- Verificar otros procesos consumidores
Quality Target: 8/10
```

### 2.5 Service Down
```
Query: "Nginx devuelve 502 Bad Gateway. ¿Dónde buscar logs y qué verificar?"
Expected:
- /var/log/nginx/error.log
- systemctl status backend-service
- netstat -tlnp | grep :8000 (upstream?)
- curl localhost:8000 (test directo)
- journalctl -u backend-service
Quality Target: 9/10
```
