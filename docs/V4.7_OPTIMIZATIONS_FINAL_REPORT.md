## ⚠️ LECCIONES APRENDIDAS

### Problema 1: `--swap-space` no soportado
**Causa:** vLLM 0.20.1 no reconoce el parámetro `--swap-space`  
**Síntoma:** Container restarting loop, error "unrecognized arguments: --swap-space 8"  
**Solución:** Removido del script de optimización  
**Commit:** Script corregido en DGX

### Problema 2: `max_model_len` excede límite del modelo
**Causa:** Qwen2.5-Coder-32B tiene `max_position_embeddings=32768`  
**Síntoma:** ValidationError al intentar usar 49152 (balanced)  
**Error:** "User-specified max_model_len (49152) is greater than the derived max_model_len (32768)"  
**Solución:** Ajustado BALANCED a 32768 en el script  
**Nota:** Para superar este límite se requiere `VLLM_ALLOW_LONG_MAX_MODEL_LEN=1` (NO RECOMENDADO)

### Problema 3: Comentarios en comando YAML
**Causa:** Docker Compose no parsea comentarios `#` en medio de `command: >` multiline  
**Síntoma:** "invalid command line string" al aplicar optimización  
**Solución:** Removidos comentarios del comando YAML generado

## 🛡️ SISTEMA DE ROLLBACK

### Backups automáticos
Cada ejecución de `optimize-vllm-performance.sh` crea:

```bash

docker compose down
cp backups/docker-compose.yml.20260505_190328 docker-compose.yml

```
