## 🔧 TROUBLESHOOTING

### Problema: Contenedor no arranca

**Síntomas:**
```bash
docker compose ps
# STATUS = Exited (1)
```

**Solución:**
```bash
# Ver logs de error
docker compose logs vllm-coder

# Errores comunes:
# 1. "CUDA out of memory" → bajar gpu_memory_utilization a 0.70
# 2. "Model not found" → verificar path /models/qwen2.5-coder-32b-awq
# 3. "Port 8001 already in use" → matar proceso en 8001
```

### Problema: vLLM lento (>30s)

**Síntomas:**
- Timeout en health_check
- Respuestas >30s

**Diagnóstico:**
```bash
# Ver métricas GPU
nvidia-smi dmon -s u -c 1

# Si GPU util <50%, problema es memory bandwidth (ARM64)
```

**Solución:**
1. Verificar AWQ quantization activa
2. Considerar reducir max_model_len de 131K a 64K
3. Fallback a Together.ai (degradation chain ya configurada)

### Problema: Ollama afectado

**Síntomas:**
- Ollama responde lento cuando vLLM activo
- OOM errors en Ollama

**Solución:**
```bash
# 1. Reiniciar Ollama
docker restart ollama

# 2. Bajar gpu_memory_utilization de vLLM a 0.70
docker compose down
# Editar docker-compose.yml: --gpu-memory-utilization 0.70
docker compose up -d

# 3. Usar vLLM on-demand (apagar cuando no se use)
docker compose stop
```
